# 看得清 Readable Output Promo Kit

## Canonical routing

- Primary install page: `https://clawhub.ai/plugins/openclaw-readable-output`
- Skill support page: `https://clawhub.ai/sheygoodbai/readable-output`
- Public proof/demo: GitHub repo `docs/demo`

Rule:

- All external posts should point to the ClawHub plugin page first.
- The demo exists to reduce hesitation for users who do not have local OpenClaw yet.
- GitHub and demo copy should always send install intent back to the ClawHub plugin page.

## Positioning

一句话版本：

把 AI 的冗长术语、过度包装和隐藏风险，整理成更适合人眼扫描的可读层。

三句版本：

1. 很多 AI 回复不是没答案，而是写得太满、太绕、太像在忽悠。
2. 看得清 Readable Output 会先把结论、重点、风险和下一步拆开，再把原文折叠保留。
3. 用户可以先看懂，再决定是否相信、是否执行。

## Honest hooks

- AI 不是总在胡说，但它常常会把“不确定”写得像“已经完成”。
- 这不是再造一个模型，而是在用户眼前加一层更容易判断真假的阅读层。
- 不读用户本地数据库，不上传聊天内容，所有规则都可以公开检查。

## Audience angles

面向重度 AI 用户：

- 少看废话，先看结论、风险和下一步。

面向产品/运营/非技术用户：

- 看懂技术回复，不再被长段术语压住。

面向多语言使用者：

- 同一套可读层规则，按不同语言的阅读习惯拆分内容。

## Store copy ideas

Short description:

`Turn dense AI output into a readable layer with overview, key points, warnings, next steps, and folded original text.`

Three proof bullets:

- Local-first readability layer for long AI output
- Keeps original wording folded underneath for verification
- Language-aware structure for English, Chinese, Japanese, Korean, Arabic, and Hebrew

## Clean promotion checklist

- Do not claim medical-grade or scientific certainty beyond the cited W3C layout guidance.
- Do not imply access to local databases, hidden telemetry, or private training.
- Do not fake reviews, installs, or “official endorsement”.
- Ask real users for before/after screenshots and quote only with permission.
