# 看得清 Readable Output Promo Kit

## Canonical routing

- Primary install page: `https://clawhub.ai/plugins/openclaw-readable-output`
- Skill support page: `https://clawhub.ai/sheygoodbai/readable-output`
- Web overlay route: Chrome extension in `browser-companion/openclaw-tool-overlay`
- Public proof/demo: GitHub repo `docs/demo`

Rule:

- The ClawHub skill can be used for discovery and homepage/search exposure, but it must provide real readability value instead of acting like a pure ad.
- All external posts should point to the ClawHub plugin page first.
- The demo exists to reduce hesitation for users who do not have local OpenClaw yet.
- GitHub and demo copy should always send install intent back to the ClawHub plugin page.
- The browser companion should be described as the honest web-only path, not as a secret plugin capability.

## Positioning

一句话版本：

把 AI 的冗长术语、过度包装和隐藏风险，整理成更适合人眼扫描的可读层。

三句版本：

1. 很多 AI 回复不是没答案，而是写得太满、太绕、太像在忽悠。
2. 看得清 Readable Output 会先把结论、重点、风险和下一步拆开，再把原文折叠保留。
3. 用户可以先看懂，再决定是否相信、是否执行。

品类句：

不是“再总结一次”，而是给 AI 输出加一个更利于判断真伪和行动风险的可读层。

## Honest hooks

- AI 不是总在胡说，但它常常会把“不确定”写得像“已经完成”。
- 这不是再造一个模型，而是在用户眼前加一层更容易判断真假的阅读层。
- 不读用户本地数据库，不上传聊天内容，所有规则都可以公开检查。

## Audience angles

面向重度 AI 用户：

- 少看废话，先看结论、风险和下一步。

面向产品/运营/非技术用户：

- 看懂技术回复，不再被长段术语压住。

面向多语言使用者：

- 同一套可读层规则，按不同语言的阅读习惯拆分内容。

## ClawHub skill role

Skill 页面适合承担三件事：

- 搜索发现：让用户通过 `readable output`、`AI output formatter`、`术语改写`、`排版优化` 一类词找到我们。
- 首页曝光：因为 skill 入口更容易被逛市场的人看见，所以标题和首屏摘要必须把价值说清楚。
- 低门槛试用：skill 本身就能先交付一次“更好读”的重写体验，再把重度用户自然引导到 plugin。

不适合做的事：

- 纯广告页
- 暗示隐藏权限
- 夸大为“官方推荐”或“保送 staff pick”
- 诱导式“刷安装”“刷评价”

## Store copy ideas

Title:

`看得清 Readable Output`

Short description:

`Turn dense AI output into a clearer reading layer with overview, key points, warnings, next steps, and folded original text.`

Opening paragraph:

`Readable Output helps users understand long AI answers before they get overwhelmed by jargon, certainty theater, or dense formatting. It surfaces the conclusion, key points, risks, and next action in a cleaner layout while keeping the original wording available underneath.`

Three proof bullets:

- Local-first readability layer for long AI output
- Keeps original wording folded underneath for verification
- Language-aware structure for English, Chinese, Japanese, Korean, Arabic, and Hebrew

Why this instead of a generic summarizer:

- It preserves an audit path by keeping the raw wording close.
- It highlights risk and uncertainty instead of smoothing them away.
- It is designed as a readability layer, not a style-heavy rewrite toy.

## Review-friendly trust copy

- Local-first settings storage only
- No user local database access
- No hidden telemetry
- No silent cloud upload claims
- Original wording remains available for verification
- Public repo and install path stay aligned

## Clean promotion checklist

- Do not claim medical-grade or scientific certainty beyond the cited W3C layout guidance.
- Do not imply access to local databases, hidden telemetry, or private training.
- Do not fake reviews, installs, or “official endorsement”.
- Ask real users for before/after screenshots and quote only with permission.

## Unsafe phrasing to reject

- `官方推荐`
- `保送 staff pick`
- `保证下载量`
- `刷安装`
- `刷好评`
- `自动读取用户本地数据库`
- `后台上传聊天内容`
- `绕过审核`

## Real growth loop

- Let the skill create the first moment of value.
- Route serious install intent to one canonical ClawHub plugin page.
- Ask real users for concrete before/after feedback in their own language.
- Keep GitHub, ClawHub, and screenshots synchronized so the same claims appear everywhere.
